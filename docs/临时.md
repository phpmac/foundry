# SH.sol 合约分析

## 部署时创建的合约数量

部署该合约会创建 **5个合约**:

1. **SH 主合约** - `AbsToken`/`SH` 合约本身
2. **TokenDistributor 合约** (第268行) - 用于 WETH 代币的分发
3. **ETHDistributor 合约** (第297行) - 用于 ETH 的分发
4. **WETH 交易对** (第244行) - 通过 PancakeSwap Factory 创建的 SH/WETH 交易对
5. **USDT 交易对** (第248行) - 通过 PancakeSwap Factory 创建的 SH/USDT 交易对

## 构造函数参数说明 (9个参数)

| 参数 | 类型 | 当前值 | 用途 |
|------|------|--------|------|
| **RouterAddress** | address | `0x10ED43...24E` | PancakeSwap V2 路由合约地址 (BSC主网) |
| **USDTAddress** | address | `0x55d398...7955` | USDT 代币地址 (BSC主网) |
| **Name** | string | `梭哈` | 代币名称 |
| **Symbol** | string | `梭哈` | 代币符号 |
| **Decimals** | uint8 | `18` | 代币精度 (小数位数) |
| **Supply** | uint256 | `21000000` | 总供应量 (实际值为 21,000,000 * 10^18) |
| **ReceiveAddress** | address | `0x47db1...7D1A` | 接收全部代币的地址 (添加白名单) |
| **FundAddress** | address | `0xe4C67...82E4` | 资金接收地址1 (收取销售费, 添加白名单) |
| **Fund2Address** | address | `0xef0E3...7cdE` | 资金接收地址2 (收取部分销售费, 添加白名单) |

### 关键用途说明

1. **RouterAddress** - 用于获取 WETH 地址、Factory 地址，以及执行代币交换和添加流动性操作
2. **USDTAddress** - 用于创建第二个交易对 (Token/USDT)
3. **ReceiveAddress** - 全部代币初始铸造到此地址，自动加入费用白名单
4. **FundAddress / Fund2Address** - 销售费用分配地址

## 销售手续费分配

没有正好 **7%** 的手续费，实际是 **7.2%** (720 基点):

| 费用项 | 基点 | 百分比 | 接收地址/用途 |
|--------|------|--------|---------------|
| `_sellLPFee` | 100 | 1% | 自动添加流动性 (LP -> fundAddress) |
| `_sellFundFee` | 300 | 3% | **fundAddress** |
| `_sellFund2Fee` | 20 | 0.2% | **fund2Address** |
| `_sellNFTFee` | 200 | 2% | NFT 持有者分红 |
| `_sellHoldFee` | 100 | 1% | 代币持有者分红 |
| **总计** | **720** | **7.2%** | |

## FundAddress vs Fund2Address 区别

| 对比项 | FundAddress (主要资金地址) | Fund2Address (次要资金地址) |
|--------|---------------------------|----------------------------|
| **销售费收益** | 3% (`_sellFundFee`) | 0.2% (`_sellFund2Fee`) |
| **LP 流动性添加** | LP 代币发到此地址 (src/sh_jmb/SH.sol:727) | 无 |
| **团队推荐剩余** | 未分配完的推荐奖励归入此地址 (src/sh_jmb/SH.sol:1218) | 无 |
| **提取 ETH 权限** | 支持 | 不支持 |
| **提取代币权限** | 支持 | 支持 |
| **比例** | 约 **93.75%** (3/3.2) | 约 **6.25%** (0.2/3.2) |

### 总结

- 两者比例为 **15:1** (3% vs 0.2%)
- **FundAddress** 是主要受益者，享有更多权限和收益来源
